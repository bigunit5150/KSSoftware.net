🏢 Multi-Tenancy Architecture
  • Kafka: Dedicated topic per customer for ordering & isolation
  • Cassandra: Partition key includes customer_id for data isolation
  • S3: Prefix-based isolation (/customer-id/...)
  • Detection Workers: Multi-tenant with resource quotas & priority queues
  • Rate Limiting: Per-customer API throttling to prevent noisy neighbors
🎯 Key Design Principles
  • Tiered Detection: 70% of emails decided in <200ms with cheap checks; expensive AI only when needed
  • Horizontal Scalability: All services stateless; scale by adding workers
  • Replay Capability: Kafka retention enables debugging, reprocessing, analytics
  • Auditability: Every decision logged with explanation (MQL rule that triggered, threat score, etc.)
  • Resource Isolation: Customer A's attack burst cannot impact Customer B's latency
  • Cost Optimization: Storage tiering, spot instances for batch, rightsized compute
